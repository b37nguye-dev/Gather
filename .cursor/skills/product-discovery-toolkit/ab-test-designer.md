# A/B Test Designer (Reference)

Statistically sound experiments with hypotheses and sample size.

## Process

1. Check product.md, personas.md, company.md
2. Gather: change, baseline metric (required), MDE, traffic volume
3. Define hypothesis: If we [change], then [metric] will [improve] because [reason]
4. Design variants: Control vs. Treatment (minimal, testable)
5. Calculate sample size (baseline, MDE, 95% sig, 80% power)
6. Define Primary, Secondary, Guardrail metrics
7. Create decision framework (what to do with each outcome)

## Output Structure

- Context (baseline, priority, persona)
- Hypothesis (If/Then/Because)
- Variants (Control, Treatment)
- Sample size (baseline, MDE, required N, estimated duration)
- Metrics tables (Primary, Secondary, Guardrails)
- Segmentation plan
- Decision framework table
- Pre-launch checklist

## Framework

Hypothesis-driven experimentation. One primary metric. Pre-register decisions. Set guardrails.
