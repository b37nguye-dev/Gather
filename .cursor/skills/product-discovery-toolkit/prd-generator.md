# PRD Generator (Reference)

Structured PRDs with problem, evidence, success criteria, and V/U/F/V risk assessment.

## Process

1. Check product.md, personas.md, company.md, competitors.md
2. Clarify problem (who, what, evidence)
3. Define success criteria: Lagging (post-launch) + Leading (pre-launch) indicators
4. Check dependencies (Feature, Team, External) — flag critical path
5. Solution exploration with 2–3 user stories
6. Risk assessment: Value, Usability, Feasibility, Viability
7. Open questions with validation experiments

## Output Structure

- Context, Problem, Evidence (Validated vs. Assumed)
- Success criteria tables (Lagging + Leading)
- Proposed solution + example user stories
- Non-goals
- Dependencies + Critical path
- Risks table (Type, Impact, Mitigation)
- Open questions with "How to Validate"
- Sign-off checklist

## Framework

Marty Cagan V/U/F/V: Address biggest risks before building. Don't invent metrics — use PLACEHOLDER.
