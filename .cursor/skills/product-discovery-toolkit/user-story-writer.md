# User Story Writer (Reference)

INVEST user stories with BDD acceptance criteria.

## Process

1. Check personas.md, product.md
2. Gather feature details (capability, persona, user goal)
3. Format: As a [persona], I want [action], So that [benefit]
4. Validate against INVEST (Independent, Negotiable, Valuable, Estimable, Small, Testable)
5. Write acceptance criteria: Given/When/Then
6. Identify edge cases

## Output Structure

- Context (persona, JTBD, related feature)
- Story (As a / I want / So that)
- INVEST checklist
- Acceptance criteria (AC1, AC2, AC3 — Given/When/Then)
- Edge cases
- Technical notes, Out of scope

## Framework

INVEST (Bill Wake) + BDD Given/When/Then. Connect "So that" to persona goals.
